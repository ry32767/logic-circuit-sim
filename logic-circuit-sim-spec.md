## 1. 概要

### 目的

論理回路をドラッグ＆ドロップで自由に組み立て、INPUT を切り替えるだけで信号の流れがリアルタイムにアニメーションし、論理回路の動作を直観的に理解できる Web シミュレーター。

### ターゲットユーザー

論理回路を学びたい人全般（学生・プログラマー志望者・趣味で学ぶ社会人）。専門知識なしでも遊びながら理解できることを最優先とする。

### 一言コンセプト

> 「回路を繋いでスイッチを押すだけで、信号が光って流れる論理回路シミュレーター」

## 4. データモデル

### 型定義（`src/types/circuit.ts`）

```ts
// ゲートの種類
type GateType =
  | 'AND' | 'OR' | 'NOT' | 'XOR'
  | 'NAND' | 'NOR' | 'XNOR' | 'BUFFER'
  | 'INPUT' | 'OUTPUT'
  | 'SR_LATCH' | 'D_FF' | 'CLK';

// ポート（入出力の接続点）
interface Port {
  id: string;
  gateId: string;
  type: 'input' | 'output';
  index: number;         // 複数ポートのうち何番目か
  position: { x: number; y: number };  // キャンバス上の絶対座標
}

// ゲート
interface Gate {
  id: string;
  type: GateType;
  x: number;             // キャンバス上のX座標
  y: number;             // キャンバス上のY座標
  // INPUT スイッチの現在値
  inputValue?: boolean;
  // CLK の周期（ms）
  clockInterval?: number;
  // SR ラッチ / D-FF の内部状態（Zustand ストアで管理）
  memoryState?: boolean;
}

// 配線
interface Wire {
  id: string;
  fromGateId: string;
  fromPortIndex: number;
  toGateId: string;
  toPortIndex: number;
}

// 回路全体（JSON エクスポート/インポートの単位）
interface Circuit {
  version: '1.0';
  gates: Gate[];
  wires: Wire[];
}
```

## 5. 回路評価ロジック

### 基本方針

- `evaluate(gates, wires, memoryStates)` を純粋関数として実装
- 各ゲートの出力値を DFS（深さ優先探索）で計算
- **循環参照の検出**：訪問済みセットで無限ループを防ぐ
- メモリ素子（SR ラッチ・D-FF）の状態は **Zustand ストアで保持**し、評価関数には外から渡す

### ゲートの評価ルール

| ゲート | 評価式 |
|---|---|
| AND | `a && b` |
| OR | `a \|\| b` |
| NOT | `!a` |
| XOR | `a !== b` |
| NAND | `!(a && b)` |
| NOR | `!(a \|\| b)` |
| XNOR | `a === b` |
| BUFFER | `a` |
| INPUT | `gate.inputValue` |
| OUTPUT | `a`（表示のみ） |

### SR ラッチの評価

```
S=1, R=0 → Q=1（セット）
S=0, R=1 → Q=0（リセット）
S=0, R=0 → Q=前回の状態（保持）
S=1, R=1 → 禁止状態（Q=0 として扱い警告表示）
```

### D フリップフロップの評価

```
CLK の立ち上がりエッジ（0→1）を検出したとき → Q = D の値を取込
CLK が変化しないとき → Q = 前回の状態を保持
```

## 6. 信号アニメーション仕様

- ワイヤーの色：HIGH = `#00e676`（緑）、LOW = `#334155`（グレー）
- HIGH 時のグロー：`filter: drop-shadow(0 0 4px #00e676)`
- 信号変化時：CSS transition `stroke 0.15s ease` でなめらかに色変化
- INPUT 切り替え時：変化した信号が末端の OUTPUT まで伝播するアニメーション（0.1s 遅延 × 経路の深さ）
